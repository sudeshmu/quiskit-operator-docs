    API Reference¶

    Complete reference documentation for all Qiskit Operator Custom Resource Definitions (CRDs).

    Custom Resources¶

    Qiskit Operator extends Kubernetes with four custom resources:

    • QiskitJob

      Execute quantum circuits on various backends

      quantum.io/v1

      View Reference

    • QiskitBackend

      Configure and manage quantum backends

      quantum.io/v1

      View Reference

    • QiskitSession

      Manage IBM Quantum Runtime sessions

      quantum.io/v1

      View Reference

    • QiskitBudget

      Control costs and enforce quotas

      quantum.io/v1

      View Reference

    Quick Reference¶

    QiskitJob¶

    Execute quantum circuits on quantum backends.

    apiVersion: quantum.io/v1
    kind: QiskitJob

    apiVersion: quantum.io/v1
kind: QiskitJob
metadata:
  name: example-job
spec:
  backend:
    type: local_simulator
  circuit:
    source: inline
    code: |
      from qiskit import QuantumCircuit
      qc = QuantumCircuit(2)
      qc.h(0)
      qc.cx(0, 1)
      qc.measure_all()
  execution:
    shots: 1024
  output:
    type: configmap
    location: results

    Key Fields: - backend: Backend configuration - circuit: Circuit code and source - execution: Execution parameters - output: Result storage

    Full Reference

    QiskitBackend¶

    Configure quantum backend resources.

    apiVersion: quantum.io/v1
    kind: QiskitBackend

    apiVersion: quantum.io/v1
kind: QiskitBackend
metadata:
  name: ibm-brisbane
spec:
  type: ibm_quantum
  name: ibm_brisbane
  credentials:
    secretRef:
      name: ibm-credentials
  default: false

    Full Reference

    QiskitSession¶

    Manage IBM Quantum Runtime sessions.

    apiVersion: quantum.io/v1
    kind: QiskitSession

    apiVersion: quantum.io/v1
kind: QiskitSession
metadata:
  name: vqe-session
spec:
  backend:
    type: ibm_quantum
    name: ibm_brisbane
  maxTime: 3600
  mode: dedicated

    Full Reference

    QiskitBudget¶

    Define cost constraints and quotas.

    apiVersion: quantum.io/v1
    kind: QiskitBudget

    apiVersion: quantum.io/v1
kind: QiskitBudget
metadata:
  name: team-budget
spec:
  limit: "$1000.00"
  period: monthly
  alerts:
    - threshold: 80
      channels: ["slack"]

    Full Reference

    Common Patterns¶

    Inline Circuit¶

    circuit:
  source: inline
  code: |
    from qiskit import QuantumCircuit
    qc = QuantumCircuit(2, 2)
    qc.h(0)
    qc.cx(0, 1)
    qc.measure([0, 1], [0, 1])

    ConfigMap Circuit¶

    circuit:
  source: configmap
  configMapRef:
    name: my-circuit
    key: circuit.py

    Git Repository Circuit¶

    circuit:
  source: git
  gitRef:
    repository: https://github.com/org/circuits
    branch: main
    path: circuits/bell.py

    Cost-Optimized Backend Selection¶

    backend:
  type: ibm_quantum
backendSelection:
  weights:
    cost: 0.70
    queueTime: 0.20
    capability: 0.10
  fallbackToSimulator: true

    Session-Based Execution¶

    spec:
  backend:
    type: ibm_quantum
  session:
    name: my-session
    maxTime: 3600
    mode: dedicated

    Status Conditions¶

    All resources provide status conditions for monitoring:

    status:
  phase: Completed
  conditions:
    - type: Validated
      status: "True"
      reason: CircuitValid
      message: Circuit validation successful
    - type: BackendSelected
      status: "True"
      reason: OptimalBackend
      message: Selected ibm_brisbane

    kubectl Commands¶

    Create Resources¶

    kubectl apply -f job.yaml
kubectl create -f budget.yaml

    List Resources¶

    kubectl get qiskitjobs
kubectl get qiskitbackends
kubectl get qiskitsessions
kubectl get qiskitbudgets

    Describe Resources¶

    kubectl describe qiskitjob my-job
kubectl describe qiskitbudget team-budget

    Delete Resources¶

    kubectl delete qiskitjob my-job
kubectl delete qiskitbudget team-budget

    Watch Resources¶

    kubectl get qiskitjob my-job -w
kubectl get qiskitjobs -w

    Validation¶

    All resources are validated before admission:

    • Schema validation (structural)
    • Semantic validation (business logic)
    • Circuit syntax validation
    • Budget enforcement
    • RBAC authorization

    API Versioning¶

    Current API version: v1

    Version Policy: - v1: Stable, production-ready - v1beta1: Pre-release, may change - v1alpha1: Experimental, unstable
